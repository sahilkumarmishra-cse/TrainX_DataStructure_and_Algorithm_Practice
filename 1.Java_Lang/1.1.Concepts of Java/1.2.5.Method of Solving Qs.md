# Methods of Solving DSA Problems

## Q: Why do we need different approaches to solve DSA problems?

=> A problem can usually be solved in multiple ways.

Some solutions are simple but slow.

Some solutions are faster but require deeper thinking, additional data structures, or mathematical observations.

The main goal in DSA is:

```text
Solve the problem correctly
↓
Reduce Time Complexity
↓
Reduce Space Complexity
↓
Make the solution scalable
```

---

## Q: What is the general progression of DSA problem-solving?

```text
1. Brute Force Approach

        ↓

2. Better Approach

        ↓

3. Optimal Approach

        ↓

4. Advanced Optimized Approach
   (when required)
```

---

# 1. Brute Force Approach

## Q: What is Brute Force?

=> Brute Force is the most straightforward and naive method of solving a problem.

It checks every possible case without trying to optimize.

Think of it as:

```text
"Try everything until the answer is found."
```

---

## Characteristics

```text
-> Easiest to understand

-> Easiest to implement

-> Usually first solution that comes to mind

-> Often inefficient

-> Higher Time Complexity

-> Useful for understanding the problem
```

---

## Example 1: Find Maximum Element

Array:

```java
int[] arr = {5, 2, 9, 1, 7};
```

Brute Force:

```java
for(int i = 0; i < arr.length; i++) {

    boolean largest = true;

    for(int j = 0; j < arr.length; j++) {

        if(arr[j] > arr[i]) {
            largest = false;
            break;
        }
    }

    if(largest) {
        System.out.println(arr[i]);
    }
}
```

Time Complexity:

```text
O(n²)
```

---

## Example 2: Two Sum Problem

Problem:

```text
Find two numbers whose sum equals target.
```

Array:

```java
[2,7,11,15]
```

Target:

```java
9
```

Brute Force:

```java
for(int i=0;i<n;i++) {

    for(int j=i+1;j<n;j++) {

        if(arr[i] + arr[j] == target) {

            System.out.println(i + " " + j);
        }
    }
}
```

Time Complexity:

```text
O(n²)
```

Space Complexity:

```text
O(1)
```

---

## When to Use Brute Force?

```text
-> To understand the problem

-> To get a correct solution first

-> When input size is small

-> During interviews as a starting point
```

---

## Drawbacks

```text
-> Slow

-> Not scalable

-> Fails for large inputs

-> Often exceeds time limits
```

---

# 2. Better Approach

## Q: What is a Better Approach?

=> A Better Approach improves the brute-force solution using observations, sorting, hashing, or extra data structures.

It is not necessarily the best possible solution but significantly improves performance.

---

## Characteristics

```text
-> Faster than brute force

-> Uses extra memory sometimes

-> Reduces unnecessary work

-> Common interview solution
```

---

## Example: Two Sum

Brute Force:

```text
O(n²)
```

Better Approach using HashMap:

```java
HashMap<Integer,Integer> map =
new HashMap<>();

for(int i=0;i<arr.length;i++) {

    int complement =
    target - arr[i];

    if(map.containsKey(complement)) {

        System.out.println(
        map.get(complement)
        + " " + i);
    }

    map.put(arr[i],i);
}
```

Time Complexity:

```text
O(n)
```

Space Complexity:

```text
O(n)
```

---

## Improvement

```text
Brute Force  -> O(n²)

Better       -> O(n)
```

Huge improvement.

---

## Common Techniques Used

```text
-> HashMap

-> HashSet

-> Sorting

-> Prefix Sum

-> Extra Arrays

-> Frequency Counting
```

# 3. Optimal Approach

## Q: What is an Optimal Approach?

=> The Optimal Approach is the best practical solution in terms of time and space complexity.

It uses deep observations, mathematical properties, or advanced algorithms.

---

## Characteristics

```text
-> Fastest possible solution

-> Interview preferred solution

-> Production-ready solution

-> Requires strong problem-solving skills
```

---

## Example: Maximum Element

Brute Force:

```text
O(n²)
```

Optimal:

```java
int max = arr[0];

for(int i=1;i<arr.length;i++) {

    if(arr[i] > max) {
        max = arr[i];
    }
}
```

Time Complexity:

```text
O(n)
```

Space Complexity:

```text
O(1)
```

---

## Example: Searching

Brute Force:

```java
Linear Search
```

Complexity:

```text
O(n)
```

Optimal:

```java
Binary Search
```

Complexity:

```text
O(log n)
```

---

## Example: Duplicate Detection

Brute Force:

```text
O(n²)
```

Using HashSet:

```text
O(n)
```

Using Sorting:

```text
O(n log n)
```

Choose based on constraints.

# 2. Two Pointer Technique

## Q: What is Two Pointer Technique?

=> Two Pointer is an optimization technique where two indices (pointers) move through a data structure simultaneously.

Instead of checking all possibilities using nested loops, we intelligently move pointers.

---

## Why Use Two Pointers?

Brute Force:

```text
O(n²)
```

Two Pointers:

```text
O(n)
```

Huge optimization.

---

## Basic Idea

```text
Array:

[10, 20, 30, 40, 50]

L                 R

Left Pointer
Right Pointer
```

Both pointers move according to conditions.

---

## When to Use Two Pointers?

```text
-> Sorted Arrays

-> Pair Sum Problems

-> Triplet Sum Problems

-> Removing Duplicates

-> Merging Arrays

-> Palindrome Problems

-> Container Problems

-> Partition Problems
```

---

## Types of Two Pointers

### 1. Opposite Direction

```text
L →          ← R
```

Example:

```text
Two Sum in Sorted Array
```

---

### 2. Same Direction

```text
L →
     R →
```

Example:

```text
Remove Duplicates
```

---

## Example: Two Sum Sorted Array

Array:

```text
[2,7,11,15]
```

Target:

```text
9
```

Solution:

```java
int left = 0;
int right = arr.length - 1;

while(left < right){

    int sum =
    arr[left] + arr[right];

    if(sum == target){
        break;
    }

    else if(sum < target){
        left++;
    }

    else{
        right--;
    }
}
```

Complexity:

```text
Time  : O(n)

Space : O(1)
```

---

## Industry Applications

```text
-> Database Searching

-> Memory Optimization

-> Financial Analytics

-> Log Processing

-> Streaming Systems
```

---

## Advantages

```text
-> Fast

-> Memory Efficient

-> Easy to Implement

-> Reduces Nested Loops
```

---

## Limitations

```text
-> Often requires sorted data

-> Not suitable for all problems
```

# 3. Sliding Window Technique

## Q: What is Sliding Window?

=> Sliding Window is an optimization technique used for problems involving:

```text
Subarrays

Substrings

Continuous Segments
```

Instead of repeatedly recalculating results, we reuse previous computations.

---

## Why Sliding Window?

Brute Force:

```text
Find maximum sum subarray
of size K.

O(n*k)
```

Sliding Window:

```text
O(n)
```

---

## Basic Idea

Window Size = 3

```text
[1,2,3,4,5,6]

Window 1:
[1,2,3]

Window 2:
  [2,3,4]

Window 3:
    [3,4,5]
```

Instead of recalculating:

```text
1+2+3

2+3+4

3+4+5
```

we update efficiently.

---

## Types of Sliding Window

### 1. Fixed Size Window

Size remains constant.

Example:

```text
Maximum Sum Subarray
of Size K
```

---

### 2. Variable Size Window

Window grows or shrinks.

Example:

```text
Longest Substring Without Repeating Characters
```

---

## Example: Maximum Sum Subarray

Array:

```text
[2,1,5,1,3,2]
```

K:

```text
3
```

Solution:

```java
int sum = 0;

for(int i=0;i<k;i++){
    sum += arr[i];
}

int max = sum;

for(int i=k;i<arr.length;i++){

    sum += arr[i];

    sum -= arr[i-k];

    max = Math.max(max,sum);
}
```

Complexity:

```text
Time  : O(n)

Space : O(1)
```

---

## Variable Window Example

Problem:

```text
Longest Substring Without Repeating Characters
```

Uses:

```text
HashSet
+
Sliding Window
```

Complexity:

```text
O(n)
```

---

## When to Use Sliding Window?

```text
-> Continuous subarray

-> Continuous substring

-> Maximum/Minimum range

-> Longest sequence

-> Shortest sequence

-> Fixed-size segment
```

---

## Industry Applications

```text
-> Network Traffic Analysis

-> Real-Time Monitoring

-> Video Streaming

-> Data Analytics

-> Sensor Data Processing

-> Stock Market Analysis
```

---

## Advantages

```text
-> Converts O(n²) to O(n)

-> Easy Optimization

-> Memory Efficient
```

---

## Limitations

```text
-> Works mainly on continuous ranges

-> Not suitable for arbitrary subsets
```

# 4. Greedy Algorithm

## Q: What is Greedy Algorithm?

=> A Greedy Algorithm makes the best possible choice at the current step hoping it leads to the globally optimal solution.

In simple words:

```text
Choose the best option now.
Don't worry about future decisions.
```

---

## Real Life Example

Imagine:

```text
You need to reach a destination.

At every step,
choose the shortest road.
```

That is Greedy Thinking.

---

## Characteristics

```text
-> Local Optimization

-> Fast Decision Making

-> Usually Simple

-> Doesn't Backtrack

-> Doesn't Reconsider Choices
```

---

## Greedy Strategy

```text
Current Best Choice

      ↓

Immediate Benefit

      ↓

Repeat

      ↓

Final Solution
```

---

## When Does Greedy Work?

A problem must have:

### 1. Greedy Choice Property

```text
Local optimum
leads to
global optimum
```

---

### 2. Optimal Substructure

```text
Optimal solution
contains
optimal solutions
to subproblems
```

---

## Example: Coin Change

Coins:

```text
10
5
2
1
```

Amount:

```text
18
```

Greedy:

```text
Take 10

Remaining = 8

Take 5

Remaining = 3

Take 2

Remaining = 1

Take 1
```

Answer:

```text
4 Coins
```

# Comparison of Major Patterns

| Pattern             | Main Use                  | Time Complexity            |
| ------------------- | ------------------------- | -------------------------- |
| Brute Force         | Try Everything            | O(n²), O(n³)               |
| Two Pointers        | Sorted Data, Pairs        | O(n)                       |
| Sliding Window      | Continuous Range Problems | O(n)                       |
| Hashing             | Fast Lookup               | O(n)                       |
| Binary Search       | Sorted Search Space       | O(log n)                   |
| Greedy              | Local Optimization        | Usually O(n) or O(n log n) |
| Dynamic Programming | Repeated Subproblems      | Varies                     |
