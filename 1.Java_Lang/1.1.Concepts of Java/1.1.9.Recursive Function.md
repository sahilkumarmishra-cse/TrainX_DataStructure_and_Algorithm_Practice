# Recursive Functions

Q: What is a Recursive Function (Method) in Java?

=> A recursive function is a method that calls itself directly or indirectly to solve a problem. Instead of solving the entire problem at once, recursion breaks a large problem into smaller versions of the same problem until a simple condition is reached that can be solved immediately.

=> Recursion is based on the principle:

```text
Big Problem
      ↓
Smaller Problem
      ↓
Even Smaller Problem
      ↓
Base Case Reached
      ↓
Solutions Return Back
```

=> Recursion is widely used in:

```
• Factorial Problems
• Fibonacci Series
• Binary Search
• Tree Traversal
• Graph Traversal (DFS)
• Backtracking
• Dynamic Programming
• Compiler Design
• Artificial Intelligence Algorithms
```

---

# Components of a Recursive Function

Q: What are the mandatory components of every recursive method?

=> Every recursive function must contain two essential parts:

1. Base Case

   • The stopping condition of recursion.

   • Prevents infinite recursive calls.

   • Once the base case becomes true, recursion stops.

Example:

```java
if(n <= 1)
    return 1;
```

2. Recursive Case

   • The section where the method calls itself.

   • Each recursive call should move closer to the base case.

Example:

```java
return n * factorial(n - 1);
```

Without a proper base case, recursion never terminates.

# How Recursion Works Internally

Q: How does recursion execute inside the JVM?

=> Every recursive call behaves like a completely new method call.

=> The JVM creates a new Stack Frame for each recursive call.

=> Each frame contains:

```
• Method Parameters
• Local Variables
• Operand Stack
• Return Address
```

Example:

```java
factorial(4);
```

Execution:

```text
factorial(4)
      ↓
factorial(3)
      ↓
factorial(2)
      ↓
factorial(1)
```

Base case reached.

Then values start returning backward.

# Recursive Expansion and Stack Unwinding

Q: What are the two phases of recursion?

=> Recursion executes in two major phases.

1. Recursive Expansion

   • Recursive calls continue creating stack frames.

Example:

```text
factorial(4)
factorial(3)
factorial(2)
factorial(1)
```

2. Stack Unwinding

   • Base case returns first.

   • Stack frames start getting removed.

Example:

```text
factorial(1)=1

factorial(2)=2×1=2

factorial(3)=3×2=6

factorial(4)=4×6=24
```

Final Answer:

```text
24
```

# Memory Mechanics of Recursion

Q: How does recursion use memory?

=> Recursion relies entirely on JVM Stack Memory.

=> Every recursive call creates a separate Stack Frame.

Example:

```java
factorial(3);
```

Call Stack:

```text
TOP

factorial(1)

factorial(2)

factorial(3)

BOTTOM
```

=> Number of recursive calls increases memory usage.

Formula:

```text
n Recursive Calls

=

n Stack Frames
```

Therefore recursion consumes additional stack memory.

# Types of Recursion

Q: What are the different types of recursion?

=> Java supports multiple recursion patterns.

1. Direct Recursion

   A method directly calls itself.

Example:

```java
void show() {
    show();
}
```

2. Indirect Recursion

   One method calls another method which eventually calls the original method.

Example:

```java
void A() {
    B();
}

void B() {
    A();
}
```

3. Head Recursion

   Recursive call occurs before processing.

Example:

```java
void print(int n){

    if(n==0)
        return;

    print(n-1);

    System.out.println(n);
}
```

Output:

```text
1
2
3
```

4. Tail Recursion

   Recursive call occurs as the final statement.

Example:

```java
void print(int n){

    if(n==0)
        return;

    print(n-1);
}
```

5. Tree Recursion

   Multiple recursive calls occur from a single function.

Example:

```java
fib(n-1) + fib(n-2)
```

# Common Implementation Mistakes

Q: What mistakes do programmers make while using recursion?

1. Missing Base Case

Example:

```java
void test(){
    test();
}
```

Result:

```text
StackOverflowError
```

2. Wrong Recursive Direction

Example:

```java
factorial(n+1);
```

Problem size increases instead of decreasing.

3. Excessively Deep Recursion

Example:

```java
factorial(100000);
```

May exhaust stack memory.

# StackOverflowError

Q: What is StackOverflowError?

=> StackOverflowError occurs when JVM Stack Memory becomes full.

Reasons:

```
• Missing Base Case
• Infinite Recursion
• Excessive Recursion Depth
• Improper Recursive Logic
```

Example:

```java
void test(){
    test();
}
```

Output:

```text
java.lang.StackOverflowError
```

# Tail Call Optimization (TCO)

Q: Does Java support Tail Call Optimization?

=> No.

=> Some programming languages optimize tail recursion by reusing stack frames.

=> Java does not perform Tail Call Optimization.

Therefore:

```text
1000 Recursive Calls

=

1000 Stack Frames
```

Every recursive call consumes memory.
