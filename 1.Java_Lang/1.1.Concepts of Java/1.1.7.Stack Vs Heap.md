# Java Memory Management - Complete Beginner to Advanced Notes

---

# 1. Introduction to Java Memory Management

Memory Management is the process of allocating, organizing, using, and releasing memory during program execution.

Whenever a Java program runs, data must be stored somewhere in memory. Variables, objects, arrays, methods, classes, and temporary calculations all require memory resources.

Unlike languages such as C and C++, Java automatically manages memory through the Java Virtual Machine (JVM), reducing programmer burden and minimizing memory-related errors.

Java Memory Management primarily involves:

- JVM (Java Virtual Machine)
- Stack Memory
- Heap Memory
- Method Area
- Garbage Collector
- Object References
- Stack Frames

Understanding these concepts is essential because every Java program, from a simple calculator to large enterprise applications, depends on them.

---

# 2. Why Memory Management is Important

Memory is a limited resource.

If memory is not managed properly:

- Programs become slow
- Memory leaks occur
- Applications consume excessive RAM
- System performance decreases
- Applications may crash

Proper memory management provides:

- Efficient RAM utilization
- Better performance
- Automatic cleanup of unused memory
- Reduced programmer effort
- Improved application stability
- Better scalability

---

# 3. JVM Overview

Java programs do not run directly on the operating system.

Execution Flow:

```text
Java Source Code (.java)
            ↓
      Java Compiler
         (javac)
            ↓
      Bytecode (.class)
            ↓
           JVM
            ↓
      Machine Code
            ↓
        Execution
```

The JVM creates and manages memory areas required for program execution.

---

# 4. JVM Memory Structure

The JVM divides memory into multiple logical regions.

```text
JVM MEMORY

├── Method Area
├── Heap Memory
├── Stack Memory
├── PC Register
└── Native Method Stack
```

For most Java developers, the most important regions are:

```text
1. Heap Memory
2. Stack Memory
```

# 5. Stack Memory

## What is Stack Memory?

Stack Memory stores temporary execution data.

It contains:

- Method Calls
- Local Variables
- Method Parameters
- Object References
- Return Addresses

Each thread receives its own private stack.

---

## Stack Working Principle

Stack follows:

```text
LIFO

Last In
First Out
```

Example:

```java
public class Demo {

    public static void main(String[] args) {

        int age = 25;

        display();
    }

    static void display() {

        int x = 10;
    }
}
```

Execution:

```text
Step 1

main() called

STACK

----------------
main()
age = 25
----------------

Step 2

display() called

STACK

----------------
display()
x = 10
----------------
main()
age = 25
----------------

Step 3

display() finishes

STACK

----------------
main()
age = 25
----------------
```

The display() frame is automatically removed.

---

## Characteristics of Stack Memory

- Thread-specific
- Fast access
- Automatic allocation
- Automatic deallocation
- Stores execution data
- Organized using LIFO

---

## Advantages

- Extremely fast
- No Garbage Collection needed
- Automatically managed
- Efficient for temporary data

---

## Limitations

- Limited size
- Cannot store large objects
- Deep recursion may cause overflow

---

# 6. Heap Memory

## What is Heap Memory?

Heap Memory stores dynamically created objects.

Whenever the keyword:

```java
new
```

is used, memory is allocated inside the Heap.

Example:

```java
Student s = new Student();
```

Memory:

```text
STACK

s
|
|
v

HEAP

Student Object
```

---

## What is Stored in Heap?

Heap stores:

- Objects
- Arrays
- Class Instances
- String Objects
- Collections
- Large Data Structures

---

## Characteristics

- Shared by all threads
- Dynamic allocation
- Large memory region
- Managed by Garbage Collector
- Slower access than stack

---

## Advantages

- Large storage capacity
- Dynamic allocation
- Suitable for OOP

---

## Limitations

- Slower access
- Requires Garbage Collection
- Can suffer OutOfMemoryError

---

# 7. Stack vs Heap Memory

| Feature      | Stack Memory    | Heap Memory       |
| ------------ | --------------- | ----------------- |
| Stores       | Local Variables | Objects & Arrays  |
| Access Speed | Faster          | Slower            |
| Allocation   | Automatic       | Dynamic           |
| Sharing      | Thread Specific | Shared            |
| Size         | Smaller         | Larger            |
| Cleanup      | Automatic       | Garbage Collector |
| Lifetime     | Method Based    | Reference Based   |

# 8. Objects and References

Many beginners think objects are stored directly in variables.

This is incorrect.

Example:

```java
Student s = new Student();
```

Actual Memory:

```text
STACK

s
|
|
v

HEAP

Student Object
```

The variable:

```java
s
```

does not contain the object.

It contains a reference (memory address) pointing to the object.

---

# 9. What Happens During Object Creation?

Example:

```java
Student s = new Student();
```

JVM performs:

Step 1

```text
Allocate memory in Heap
```

Step 2

```text
Initialize instance variables
```

Step 3

```text
Call constructor
```

Step 4

```text
Store reference in Stack
```

Final Memory:

```text
STACK

s
|
|
v

HEAP

Student
{
id = 0
name = null
}
```

# 10. Stack Frames

Every method call creates a Stack Frame.

Example:

```java
main()
{
    add();
}
```

Execution:

```text
STACK

-----------
add()
-----------

-----------
main()
-----------
```

When add() finishes:

```text
STACK

-----------
main()
-----------
```

The frame is removed automatically.

---

# 11. Components of a Stack Frame

Each Stack Frame contains:

```text
1. Local Variable Table
2. Operand Stack
3. Return Address
```

---

## Local Variable Table

Stores:

- Parameters
- Local Variables

Example:

```java
void add(int a, int b)
{
    int sum = a + b;
}
```

Suppose:

```java
add(5,10);
```

Table:

```text
a = 5
b = 10
sum = 15
```

---

## Operand Stack

Used by JVM during calculations.

Example:

```java
int c = a + b;
```

JVM internally:

```text
Push a
Push b
Add
Store result
```

Operand Stack:

```text
Top
-----
b
a
-----
```

After addition:

```text
15
```

is stored.

---

## Return Address

Stores location where execution must return after the method completes.

Example:

```java
main()
{
   add();
   print();
}
```

After add() completes, JVM knows execution should continue with:

```java
print();
```

because the return address was stored.

---

# 12. Method Area

Method Area stores:

- Class Metadata
- Method Information
- Static Variables
- Runtime Constant Pool

Example:

```java
class Student {

    static int count;
}
```

The static variable:

```java
count
```

is stored in Method Area.

Not Heap.

Not Stack.

---

# 13. Garbage Collection

## What is Garbage Collection?

Garbage Collection is the automatic process of removing unreachable objects from Heap Memory.

Example:

```java
Student s = new Student();

s = null;
```

Now:

```text
No reference exists
```

Object becomes eligible for Garbage Collection.

---

## Important Rule

Eligible for Garbage Collection

≠

Immediately Deleted

The JVM decides when to actually remove it.

---

# 14. When Does an Object Become Eligible for GC?

Case 1

```java
s = null;
```

Case 2

```java
s = new Student();
s = new Student();
```

Old object loses reference.

Case 3

Method ends.

```java
void test()
{
   Student s = new Student();
}
```

After method completion:

```text
Reference destroyed
```

Object becomes unreachable.

---

# 15. Garbage Collection Process

Simplified Process:

```text
Object Created
      ↓
Object Used
      ↓
Reference Lost
      ↓
Object Becomes Unreachable
      ↓
Garbage Collector Marks It
      ↓
Memory Reclaimed
```

---

# 16. Memory Lifecycle of an Object

Example:

```java
Student s = new Student();
```

Step 1

Object created in Heap.

Step 2

Reference stored in Stack.

Step 3

Object used.

Step 4

Reference removed.

```java
s = null;
```

Step 5

Garbage Collector removes object.

# 17. Common Memory Errors

## StackOverflowError

Occurs when stack memory becomes full.

Most common cause:

Infinite recursion.

Example:

```java
void test()
{
    test();
}
```

Result:

```text
StackOverflowError
```

---

## OutOfMemoryError

Occurs when Heap Memory becomes exhausted.

Example:

```java
while(true)
{
    new Student();
}
```

Result:

```text
OutOfMemoryError
```
