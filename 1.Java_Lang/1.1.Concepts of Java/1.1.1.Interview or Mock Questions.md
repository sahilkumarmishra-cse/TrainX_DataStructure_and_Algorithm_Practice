# 1. What are Functions?

In Java, functions are called **methods**.

A method is a named block of code that performs a specific task and executes only when it is called (invoked).

Since Java is a class-based language, every method must be defined inside a class.

### General Syntax

```java
class Demo {

    returnType methodName(parameters) {

        // method body

        return value;
    }
}
```

### Example

```java
class Demo {

    static int add(int a, int b) {

        return a + b;
    }

    public static void main(String[] args) {

        int result = add(5, 3);

        System.out.println(result);
    }
}
```

### Characteristics

- Reusable
- Modular
- Improves readability
- Reduces code duplication
- Easier debugging and maintenance

---

# 2. Why Functions?

Without functions:

```java
int sum1 = 5 + 3;
int sum2 = 10 + 20;
int sum3 = 7 + 8;
```

Same logic repeated multiple times.

With functions:

```java
int add(int a, int b) {
    return a + b;
}
```

Use:

```java
add(5,3);
add(10,20);
add(7,8);
```

### Advantages

1. Code Reusability
2. Modularity
3. Better Maintenance
4. Easier Testing
5. Improved Readability
6. Reduced Complexity
7. Better Abstraction

# 4. Sequential Flow of Functions

Example:

```java
class Demo {

    static void first() {
        System.out.println("First");
    }

    static void second() {
        System.out.println("Second");
    }

    public static void main(String[] args) {

        first();

        second();
    }
}
```

### Execution Flow

```text
JVM starts
    ↓
main()
    ↓
first()
    ↓
returns to main()
    ↓
second()
    ↓
returns to main()
    ↓
program ends
```

### Important Rule

Execution always returns to the caller after method completion.

# 5. Context of Function (Scope and this Keyword)

## Scope

Scope defines where a variable is accessible.

### Local Scope

```java
void test() {

    int x = 10;
}
```

`x` exists only inside `test()`.

---

### Class Scope

```java
class Demo {

    int x = 10;
}
```

Accessible throughout the object.

---

## this Keyword

`this` refers to the current object.

Example:

```java
class Student {

    String name;

    Student(String name) {

        this.name = name;
    }
}
```

### Why Needed?

Without `this`:

```java
name = name;
```

Both refer to parameter.

With `this`:

```java
this.name = name;
```

Left side → object variable

Right side → parameter

# 6. Stack vs Heap Memory, Local Variable Table, Object Storage

## Stack Memory

Stores:

- Method calls
- Local variables
- References

Example:

```java
int x = 10;
```

Stored in stack.

---

## Heap Memory

Stores:

- Objects
- Arrays

Example:

```java
Student s = new Student();
```

Object stored in heap.

Reference `s` stored in stack.

---

## Visualization

```java
Student s = new Student();
```

```text
STACK                    HEAP

s --------->      Student Object
```

---

## Local Variable Table

Each method frame contains:

```text
Parameter values
Local variables
Temporary values
```

Example:

```java
void add(int a, int b) {

    int sum = a + b;
}
```

Local Variable Table:

```text
a = 5
b = 10
sum = 15
```

---

# 7. Stack Frames

Every method call creates a stack frame.

Example:

```java
main()
{
    add();
}
```

Execution:

```text
Stack

---------
add()
---------
main()
---------
```

When add() finishes:

```text
Stack

---------
main()
---------
```

Frame removed automatically.

Each frame contains:

- Local Variable Table
- Operand Stack
- Return Address

# 8. Call by Value vs Call by Reference

## Java is 100% Call By Value

### Primitive Example

```java
int x = 10;

change(x);
```

Method receives copy.

```java
void change(int a) {
    a = 20;
}
```

Original remains:

```text
10
```

---

### Object Example

```java
Student s = new Student();
```

Java passes copy of reference.

```java
modify(s);
```

```java
void modify(Student obj) {
    obj.name = "Rahul";
}
```

Object changes.

But:

```java
obj = new Student();
```

doesn't change original reference.

### Conclusion

Java never uses true call-by-reference.

Java always uses:

```text
Call By Value
```

including object references.

# 10. Recursive Functions

A method calling itself.

Example:

```java
int factorial(int n) {

    if(n == 1)
        return 1;

    return n * factorial(n - 1);
}
```

### Execution

```text
factorial(5)
    ↓
5 * factorial(4)
    ↓
5 * 4 * factorial(3)
    ↓
5 * 4 * 3 * factorial(2)
    ↓
5 * 4 * 3 * 2 * factorial(1)
    ↓
120
```

### Components

1. Base Case
2. Recursive Call

Without base case:

```text
StackOverflowError
```

# 11. Difference Between Parameters and Arguments

## Parameters

Variables declared in method definition.

```java
int add(int a, int b)
```

`a` and `b` are parameters.

---

## Arguments

Actual values passed.

```java
add(10, 20);
```

`10` and `20` are arguments.

### Memory View

```text
Method Definition
a, b
↑
Parameters

Method Call
10, 20
↑
Arguments
```

# 12. Method Overloading vs Method Overriding

## Method Overloading

Same method name.

Different parameter list.

```java
class MathUtil {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

### Features

- Same class
- Compile-time polymorphism
- Parameters differ

---

## Method Overriding

Subclass redefines parent method.

```java
class Animal {

    void sound() {
        System.out.println("Animal Sound");
    }
}

class Dog extends Animal {

    @Override
    void sound() {
        System.out.println("Bark");
    }
}
```

### Features

- Inheritance required
- Runtime polymorphism
- Same signature
- Same return type (or covariant)

| -------------------- | ------------------------------------------- |
| Function/Method      | Reusable block of code                      |
| Scope                | Region where variable is accessible         |
| this                 | Current object reference                    |
| Stack                | Stores method calls and local variables     |
| Heap                 | Stores objects and arrays                   |
| Stack Frame          | Memory block created per method call        |

| Parameters           | Variables in method definition              |
| Arguments            | Actual values passed                        |

This forms the foundation of how methods/functions work internally in Java, from source code level down to JVM memory and execution flow.
