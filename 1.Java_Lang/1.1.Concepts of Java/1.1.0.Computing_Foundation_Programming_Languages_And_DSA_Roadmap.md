# 2. Hardware and Software

## What is Hardware?

Hardware refers to the physical components of a computer system.

### Examples

- CPU (Central Processing Unit)
- RAM (Random Access Memory)
- Motherboard
- Hard Disk / SSD
- Keyboard
- Mouse
- Monitor
- Network Devices

### Purpose

Hardware performs computations and executes instructions provided by software.

---

## What is Software?

Software refers to a collection of programs and instructions that tell hardware what to do.

### Examples

- Windows
- Linux
- Android
- Web Browsers
- Mobile Applications
- Games
- Database Systems

### Purpose

Software provides functionality and enables users to interact with hardware.

---

## Relationship Between Hardware and Software

```text
User
  ↓
Application Software
  ↓
Operating System
  ↓
Hardware
```

Hardware cannot function effectively without software, and software cannot execute without hardware.

# 3. Evolution of Hardware

## Vacuum Tubes (1940s)

### Advantages

- First electronic computers

### Limitations

- Large size
- High heat generation
- High power consumption

---

## Transistors (1947)

### Advantages

- Smaller size
- Faster operation
- Higher reliability
- Lower power consumption

---

## Integrated Circuits (1958)

### Advantages

- Multiple transistors on a single chip
- Smaller devices
- Lower manufacturing cost

---

## Microprocessors (1971)

A microprocessor integrates the CPU onto a single chip.

### Impact

- Personal Computers
- Smartphones
- Embedded Systems
- Robotics
- IoT Devices

---

# 4. Evolution of Software

## Machine Language

Computers originally understood only binary instructions.

Example:

```text
10101010
11001100
01010101
```

### Problems

- Difficult for humans to read
- Difficult to write
- Difficult to debug

---

## Assembly Language

Introduced symbolic instructions.

Example:

```assembly
MOV A, B
ADD A, B
SUB A, B
```

### Benefits

- Easier than machine language
- Closer to hardware

### Limitation

- Hardware dependent

---

## High-Level Programming Languages

Created to improve productivity and portability.

Examples:

- FORTRAN
- COBOL
- C
- C++
- Java
- Python

# 6. Why Data Structures and Algorithms (DSA)?

## Common Misconception

Many beginners think learning a programming language is enough.

Example:

```java
for(int i = 0; i < n; i++) {
    System.out.println(i);
}
```

The program works.

However, what happens when:

```text
n = 10
```

vs

```text
n = 10,000,000
```

Efficiency becomes extremely important.

---

## Purpose of DSA

DSA helps answer:

### Time Efficiency

How fast is the program?

### Space Efficiency

How much memory does the program use?

---

## Real Industry Importance

Large companies process:

- Millions of users
- Billions of records
- Massive datasets

Efficient algorithms reduce:

- Processing time
- Memory usage
- Infrastructure cost

---

# 7. Time Complexity Overview

| Complexity | Performance           |
| ---------- | --------------------- |
| O(1)       | Excellent             |
| O(log n)   | Very Good             |
| O(n)       | Good                  |
| O(n log n) | Acceptable            |
| O(n²)      | Poor for large inputs |
| O(2ⁿ)      | Very Poor             |
| O(n!)      | Extremely Poor        |
