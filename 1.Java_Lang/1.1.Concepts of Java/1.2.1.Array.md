# 1. Array Basics & Declaration

## Q: What is an Array in Java?

=> An Array is a **linear, fixed-size, homogeneous data structure** used to store multiple values of the same data type under a single variable name.

Instead of creating multiple variables:

```java
int mark1 = 90;
int mark2 = 85;
int mark3 = 75;
int mark4 = 88;
int mark5 = 95;
```

We can use:

```java
int[] marks = {90, 85, 75, 88, 95};
```

Array Representation:

```text
Index :  0   1   2   3   4
Value : 90  85  75  88  95
```

---

## Q: Why do we need Arrays?

=> Arrays help store large collections of similar data efficiently.

Without Arrays:

```java
int student1 = 90;
int student2 = 85;
int student3 = 75;
...
```

Problems:

```text
-> Too many variables
-> Difficult to manage
-> Difficult to search
-> Difficult to sort
-> Difficult to process using loops
```

With Arrays:

```java
int[] marks = {90, 85, 75, 88, 95};
```

Benefits:

```text
-> Less code
-> Easy traversal
-> Easy searching
-> Easy sorting
-> Better memory organization
-> Fast random access
```

---

## Q: What are the characteristics of Arrays?

=> Arrays have the following properties:

### 1. Fixed Size

Once created, the size cannot be changed.

```java
int[] arr = new int[5];
```

The array will always contain exactly 5 elements.

---

### 2. Homogeneous

All elements must be of the same data type.

Valid:

```java
int[] arr = {10, 20, 30};
```

Invalid:

```java
int[] arr = {10, "Java", 30};
```

Reason:

```text
An int array can only store integers.
```

---

### 3. Index Based

Every element is accessed using an index.

```text
Index : 0 1 2 3 4
Value :10 20 30 40 50
```

Example:

```java
arr[2]
```

Output:

```text
30
```

---

### 4. Contiguous Memory Allocation

Conceptually, array elements are stored sequentially in memory.

```text
Address      Value

1000  ---->  10
1004  ---->  20
1008  ---->  30
1012  ---->  40
1016  ---->  50
```

This enables fast access.

---

```java

```

Example:

```java

```

```java
arr.length
```



# 2. Accessing & Modifying Elements

## Q: How do we access an element?

=> Using its index.

Syntax:

```java
arrayName[index]
```

Example:

```java
int[] arr = {10,20,30};

System.out.println(arr[1]);
```

Output:

```text
20
```

---

## Q: What is the time complexity of accessing an element?

```text
O(1)
```

Reason:

```text
Direct memory lookup using index.
```

---

## Q: How do we modify an element?

Example:

```java
arr[2] = 99;
```

Before:

```text
10 20 30
```

After:

```text
10 20 99
```

---

## Q: What happens when we access an invalid index?

Example:

```java
int[] arr = {10,20,30};

System.out.println(arr[5]);
```

Output:

```text
ArrayIndexOutOfBoundsException
```

Reason:

Valid indices:

```text
0 to length - 1
```

For length = 3:

```text
0,1,2
```

only.

# 3. Traversing Arrays (Looping)

## Q: What is Traversal?

=> Traversal means visiting every element of the array exactly once.

---

## Q: How do we traverse using a for loop?

```java
for(int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}
```

Advantages:

```text
-> Index available
-> Modification possible
-> Most flexible
```

Time Complexity:

```text
O(n)
```

---

## Q: How do we traverse using Enhanced For Loop?

Syntax:

```java
for(dataType variable : arrayName)
```

Example:

```java
for(int num : arr) {
    System.out.println(num);
}
```

Advantages:

```text
-> Cleaner syntax
-> Less code
-> Easy reading
```

Limitation:

```text
Cannot directly access index.
Cannot modify original elements.
```

# 4. Common Array Operations

## Searching

### Q: What is Linear Search?

=> Search elements one by one.

Example:

```text
10 20 30 40 50
```

Search:

```text
30
```

Process:

```text
10 -> No
20 -> No
30 -> Found
```

Complexity:

```text
Best Case  : O(1)
Worst Case : O(n)
```

---

### Q: What is Binary Search?

=> Binary Search repeatedly divides the search space into two halves.

Requirement:

```text
Array must be sorted.
```

Example:

```text
10 20 30 40 50
```

Search:

```text
30
```

Complexity:

```text
O(log n)
```

Much faster than Linear Search.

---

## Insertion

Q: Why is insertion costly in arrays?

=> Because elements need to be shifted.

Before:

```text
10 20 40 50
```

Insert:

```text
30
```

After:

```text
10 20 30 40 50
```

Complexity:

```text
O(n)
```

---

## Deletion

Before:

```text
10 20 30 40 50
```

Delete:

```text
30
```

After:

```text
10 20 40 50
```

Elements must shift left.

Complexity:

```text
O(n)
```

---

## Reversing

Before:

```text
10 20 30 40 50
```

After:

```text
50 40 30 20 10
```

Algorithm:

```text
Swap first and last.
Swap second and second-last.
Continue until center.
```

Complexity:

```text
O(n)
```

---

# 6. Multidimensional Arrays (Arrays of Arrays)

## Q: What is a 2D Array?

=> A 2D Array is an array whose elements are themselves arrays.

Example:

```java
int[][] matrix = {
    {1,2},
    {3,4},
    {5,6}
};
```

Visualization:

```text
1 2
3 4
5 6
```

---

## Declaration

```java
int[][] matrix = new int[3][4];
```

Meaning:

```text
3 Rows
4 Columns
```

---

## Q: What are Jagged Arrays?

=> Arrays whose rows have different lengths.

Example:

```java
int[][] jagged = new int[2][];

jagged[0] = new int[3];
jagged[1] = new int[5];
```

Visualization:

```text
Row 0 -> 3 columns

Row 1 -> 5 columns
```

---

## Q: How do we traverse a 2D Array?

Using nested loops:

```java
for(int i=0;i<matrix.length;i++) {

    for(int j=0;j<matrix[i].length;j++) {

        System.out.print(matrix[i][j] + " ");
    }
}
```

Complexity:

```text
O(rows × columns)
```

# 7. Memory Management & Limitations

## Q: Where are Arrays stored in memory?

=> Arrays are objects in Java.

Example:

```java

```

Memory Allocation:

```text
Stack

arr
 |
 |
 V

Heap

[0][0][0][0][0]
```

The variable stores a reference to the array object.
