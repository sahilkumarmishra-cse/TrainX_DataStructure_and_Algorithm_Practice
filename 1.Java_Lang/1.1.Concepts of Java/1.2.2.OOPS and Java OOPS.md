# Object-Oriented Programming System (OOPS) in Java

## Q: What is OOPS?

=> OOPS (Object-Oriented Programming System) is a programming paradigm that organizes software around Objects rather than Functions.

An Object represents a real-world entity that contains:

```text
1. State (Data / Attributes)
2. Behavior (Methods / Functions)
```

````

Example:

```text
Object: Car

State:
-> Color
-> Brand
-> Speed

Behavior:
-> Start()
-> Stop()
-> Accelerate()
```

---

## Q: Why was OOPS introduced?

=> Before OOPS, programs were mainly written using Procedural Programming.

Example:

```text
C Language
```

Problems with Procedural Programming:

```text
-> Large code becomes difficult to manage

-> Low code reusability

-> Weak data security

-> Difficult maintenance

-> Difficult real-world modeling
```

OOPS solves these problems by:

```text
-> Encapsulation

-> Inheritance

-> Polymorphism

-> Abstraction
```

# Object in Java

## Q: What is an Object?

=> An Object is an instance of a class.

It occupies memory and represents a real-world entity.

Example:

```java
class Student {

    String name;
    int age;
}

public class Main {

    public static void main(String[] args) {

        Student s1 = new Student();

        s1.name = "Rahul";
        s1.age = 20;
    }
}
```

Explanation:

```text
Student -> Class

s1 -> Object

new Student() -> Creates Object in Heap Memory
```

---

# Class in Java

## Q: What is a Class?

=> A Class is a blueprint or template used to create objects.

Example:

```java
class Student {

    String name;
    int age;

    void display() {
        System.out.println(name);
        System.out.println(age);
    }
}
```

Explanation:

```text
Class does not occupy memory for data.

Memory is allocated only when object is created.
```

---

## Real-Life Example

Blueprint:

```text
House Design
```

Actual House:

```text
Constructed House
```

Similarly:

```text
Class -> Blueprint

Object -> Actual Entity
```

---

# Class vs Object

| Feature    | Class         | Object          |
| ---------- | ------------- | --------------- |
| Definition | Blueprint     | Instance        |
| Memory     | No Memory     | Occupies Memory |
| Creation   | Declared Once | Can Create Many |
| Example    | Student       | s1, s2, s3      |

# 1. Encapsulation

## Q: What is Encapsulation?

=> Encapsulation means wrapping Data and Methods together into a single unit (Class).

Example:

```java
class Student {

    private String name;

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}
```

Explanation:

```text
Data -> Private

Access -> Through Methods
```

---

## Why Encapsulation?

Benefits:

```text
-> Data Security

-> Data Hiding

-> Better Maintenance

-> Controlled Access

-> Improved Reliability
```

---

## Real-Life Example

ATM Machine

```text
User can withdraw money

User cannot directly access bank database
```

Encapsulation works similarly.

# 2. Inheritance

## Q: What is Inheritance?

=> Inheritance allows one class to acquire properties and methods of another class.

Keyword:

```java
extends
```

---

## Example

```java
class Animal {

    void eat() {
        System.out.println("Eating");
    }
}

class Dog extends Animal {

    void bark() {
        System.out.println("Barking");
    }
}
```

Usage:

```java
Dog d = new Dog();

d.eat();

d.bark();
```

Output:

```text
Eating
Barking
```

---

## Benefits of Inheritance

```text
-> Code Reusability

-> Faster Development

-> Easy Maintenance

-> Hierarchical Design
```

---

## Types of Inheritance

### Single Inheritance

```text
A -> B
```

---

### Multilevel Inheritance

```text
A -> B -> C
```

---

### Hierarchical Inheritance

```text
       A
      / \
     B   C
```

---

### Multiple Inheritance

Not supported using Classes.

Supported using Interfaces.

# 3. Polymorphism

## Q: What is Polymorphism?

=> Polymorphism means "Many Forms".

Same method behaves differently in different situations.

---

## Types of Polymorphism

### Compile-Time Polymorphism

Method Overloading

Example:

```java
class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

---

### Runtime Polymorphism

Method Overriding

Example:

```java
class Animal {

    void sound() {
        System.out.println("Animal Sound");
    }
}

class Dog extends Animal {

    @Override
    void sound() {
        System.out.println("Dog Bark");
    }
}
```

Usage:

```java
Animal a = new Dog();

a.sound();
```

Output:

```text
Dog Bark
```

---

## Benefits

```text
-> Flexibility

-> Extensibility

-> Dynamic Behavior

-> Cleaner Design
```

---

# 4. Abstraction

## Q: What is Abstraction?

=> Abstraction means hiding implementation details and showing only essential features.

---

## Real-Life Example

Car

```text
User knows:

-> Start
-> Stop
-> Brake

User doesn't know:

-> Engine Combustion
-> Fuel Injection
-> Internal Wiring
```

---

## Achieved Through

```text
1. Abstract Class

2. Interface
```

---

# Abstract Class

## Definition

A class declared using:

```java
abstract
```

keyword.

---

## Example

```java
abstract class Animal {

    abstract void sound();
}

class Dog extends Animal {

    void sound() {
        System.out.println("Bark");
    }
}
```

---

## Rules

```text
-> Cannot create object

-> Can contain abstract methods

-> Can contain normal methods

-> Child class must implement abstract methods
```

---

# Interface

## Definition

An Interface contains behavior contracts.

---

## Example

```java
interface Vehicle {

    void start();
}

class Car implements Vehicle {

    public void start() {
        System.out.println("Car Started");
    }
}
```

---

## Why Interface?

```text
-> Achieves Abstraction

-> Supports Multiple Inheritance

-> Standardization
```

# Constructor

## Q: What is a Constructor?

=> Constructor is a special method used to initialize objects.

Rules:

```text
-> Same name as class

-> No return type

-> Called automatically
```

---

## Example

```java
class Student {

    Student() {
        System.out.println("Constructor Called");
    }
}
```

---

# Constructor Types

## Default Constructor

```java
Student() {

}
```

---

## Parameterized Constructor

```java
Student(String name) {

}
```

---

## Constructor Overloading

```java
Student() {

}

Student(String name) {

}

Student(String name, int age) {

}
```

# this Keyword

## Q: What is this?

=> Refers to current object.

Example:

```java
class Student {

    String name;

    Student(String name) {

        this.name = name;
    }
}
```

---

# super Keyword

## Q: What is super?

=> Refers to parent class object.

Example:

```java
class Animal {

    String type = "Animal";
}

class Dog extends Animal {

    void show() {
        System.out.println(super.type);
    }
}
```

---

# Static Keyword

## Q: What is static?

=> Belongs to Class rather than Object.

Example:

```java
class Student {

    static String college = "ABC";
}
```
````
