# Method Overloading in Java

## What is Method Overloading?

Method Overloading is a feature in Java that allows multiple methods within the same class to have the same name but different parameter lists.

A parameter list may differ in:

- Number of parameters
- Data types of parameters
- Order (sequence) of parameters

Method Overloading is one of the ways Java implements **Compile-Time Polymorphism (Static Polymorphism)**.

### Definition

> Method Overloading means defining multiple methods with the same name in the same class while keeping their signatures different.

Example:

```java
class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

Here, all methods are named `add()`, but their parameter lists are different.

Therefore, they are overloaded methods.

---

# Why Do We Need Method Overloading?

Suppose Java did not support method overloading.

We would have to write:

```java
printInteger(int x);

printDouble(double x);

printString(String x);
```

Problems:

- Too many method names
- Difficult to remember
- Reduced readability
- Poor API design

With Method Overloading:

```java
print(int x);

print(double x);

print(String x);
```

Benefits:

- Same operation uses the same name
- Easier to remember
- Cleaner code
- Better software design

The method name should describe the operation being performed, not the data type being used.

---

# Real-Life Analogy

Consider a calculator.

A calculator performs addition on:

```text
2 + 3
2.5 + 3.5
2 + 3 + 4
```

All operations are "addition."

Therefore, using a common method name:

```java
add()
```

makes more sense than:

```java
addInteger()
addDouble()
addThreeNumbers()
```

Method Overloading allows one operation name to work with different inputs.

# Syntax of Method Overloading

General Syntax:

```java
returnType methodName(parameterList) {

    // statements

}
```

Example:

```java
class Demo {

    void show() {
        System.out.println("No Parameters");
    }

    void show(int x) {
        System.out.println(x);
    }

    void show(String name) {
        System.out.println(name);
    }
}
```

Usage:

```java
Demo d = new Demo();

d.show();

d.show(10);

d.show("Java");
```

Output:

```text
No Parameters
10
Java
```

# Method Signature

Method Signature is the identity of a method from the compiler's perspective.

### Method Signature =

```text
Method Name + Parameter List
```

Example:

```java
void display(int x)
```

Signature:

```text
display(int)
```

Example:

```java
void show(int x)

void show(double x)
```

Signatures:

```text
show(int)

show(double)
```

Different signatures ⇒ Valid Overloading

---

## What is NOT Part of a Signature?

A method signature DOES NOT include:

- Return Type
- Access Modifier
- Method Body
- Exception List

Example:

```java
void show(int x)

int show(int x)
```

Both signatures are:

```text
show(int)
```

Therefore Java considers them identical.

---

# Rules for Method Overloading

A method can be overloaded only if its signature changes.

There are three legal ways.

---

## Rule 1: Different Number of Parameters

Example:

```java
int add(int a, int b)

int add(int a, int b, int c)
```

Signatures:

```text
add(int,int)

add(int,int,int)
```

Valid Overloading

---

## Rule 2: Different Data Types

Example:

```java
int add(int a, int b)

double add(double a, double b)
```

Signatures:

```text
add(int,int)

add(double,double)
```

Valid Overloading

---

## Rule 3: Different Order of Parameters

Example:

```java
void display(int a, String b)

void display(String b, int a)
```

Signatures:

```text
display(int,String)

display(String,int)
```

Valid Overloading

---

# Invalid Overloading Cases

---

## Case 1: Changing Only Return Type

Invalid

```java
class Demo {

    void show(int x) {
    }

    int show(int x) {
        return x;
    }
}
```

Compiler Error

Reason:

Both methods have the same signature:

```text
show(int)
```

The compiler cannot distinguish them.

---

## Case 2: Changing Only Access Modifier

Invalid

```java
class Demo {

    public void show(int x) {
    }

    private void show(int x) {
    }
}
```

Same signature.

Compiler Error.

---

## Case 3: Changing Only Method Body

Invalid

```java
void show(int x) {
}

void show(int x) {
    System.out.println(x);
}
```

Same signature.

Compiler Error.

---

# How Java Resolves Overloaded Methods

This process is called **Overload Resolution**.

When Java encounters:

```java
add(10, 20);
```

the compiler follows these steps.

### Step 1: Exact Match

Search for:

```java
add(int, int)
```

If found, execute it.

---

### Step 2: Type Promotion

If no exact match exists, Java attempts type promotion.

---

### Step 3: Compile-Time Error

If no suitable method is found, compilation fails.

Flow:

```text
Method Call
     |
     V
Exact Match?
     |
   YES ----> Execute
     |
     NO
     |
     V
Type Promotion?
     |
   YES ----> Execute
     |
     NO
     |
     V
Compile-Time Error
```

---

# Automatic Type Promotion

If Java cannot find an exact match, it automatically promotes smaller data types to larger data types.

Promotion Hierarchy:

```text
byte
 ↓
short
 ↓
int
 ↓
long
 ↓
float
 ↓
double
```

---

## Example

```java
class Demo {

    void check(long x) {
        System.out.println("Long");
    }

    void check(double x) {
        System.out.println("Double");
    }

    public static void main(String[] args) {

        Demo d = new Demo();

        d.check(5);
    }
}
```

Output:

```text
Long
```

Reason:

```text
5 → int

No check(int)

int → long
```

Therefore:

```java
check(long)
```

executes.

# Constructor Overloading

Constructors can also be overloaded.

Example:

```java
class Student {

    Student() {
        System.out.println("Default Constructor");
    }

    Student(String name) {
        System.out.println(name);
    }

    Student(String name, int age) {
        System.out.println(name + " " + age);
    }
}
```

Usage:

```java
Student s1 = new Student();

Student s2 = new Student("Rahul");

Student s3 = new Student("Rahul", 20);
```

Benefits:

- Multiple ways to create objects
- Flexible object initialization

---

# Method Overloading vs Method Overriding

| Feature      | Method Overloading | Method Overriding |
| ------------ | ------------------ | ----------------- |
| Class        | Same Class         | Parent + Child    |
| Method Name  | Same               | Same              |
| Parameters   | Different          | Same              |
| Binding      | Compile-Time       | Runtime           |
| Polymorphism | Static             | Dynamic           |

# Advantages of Method Overloading

### 1. Improved Readability

```java
print(10);

print("Java");
```

is cleaner than:

```java
printInteger(10);

printString("Java");
```

---

### 2. Better API Design

Developers remember fewer method names.

---

### 3. Code Reusability

Same operation works for multiple inputs.

---

### 4. Compile-Time Polymorphism

Provides static polymorphism.

---

### 5. Flexibility

Handles different input types efficiently.

---

# Disadvantages of Method Overloading

### 1. Code Confusion

Too many overloaded methods can reduce readability.

Example:

```java
show(int,double)

show(double,int)

show(float,long)
```

---

### 2. Ambiguous Calls

Sometimes the compiler may become confused.

Example:

```java
show(null);
```

when multiple reference-type overloads exist.

---

### 3. Type Promotion Issues

Automatic promotion may call an unexpected method.

# Summary

Method Overloading allows multiple methods with the same name to exist in the same class, provided their parameter lists differ.

A valid overload must differ in:

1. Number of Parameters
2. Parameter Data Types
3. Parameter Order

Java resolves overloaded methods using:

1. Exact Match
2. Type Promotion
3. Compile-Time Error

Method Overloading is an example of Compile-Time (Static) Polymorphism and is widely used in Java libraries such as:

- System.out.println()
- Math.max()
- Constructors
- Collection APIs
- Enterprise Applications

It improves readability, flexibility, maintainability, and software design.
