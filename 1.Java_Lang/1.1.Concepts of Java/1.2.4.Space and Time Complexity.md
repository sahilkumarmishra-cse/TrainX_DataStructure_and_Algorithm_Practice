# Time Complexity and Space Complexity

## Q: What is Complexity Analysis?

=> Complexity Analysis is the process of measuring how efficient an algorithm is in terms of:

```text
1. Time Complexity
2. Space Complexity
```

It helps programmers predict how an algorithm will perform as the input size grows.

---

## Why Do We Need Complexity Analysis?

Consider two programs:

Program 1:

```java
for(int i = 0; i < n; i++) {
    System.out.println(i);
}
```

Program 2:

```java
for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        System.out.println(i + " " + j);
    }
}
```

Both programs work correctly.

But:

```text
Program 1 → Faster

Program 2 → Slower
```

As input size increases:

```text
n = 10

Program 1 → 10 operations

Program 2 → 100 operations
```

n = 1000

```text
Program 1 → 1,000 operations

Program 2 → 1,000,000 operations
```

Complexity Analysis helps identify which solution is more efficient.

---

# What is Time Complexity?

## Definition

=> Time Complexity measures how the running time of an algorithm grows with respect to the input size.

It does NOT measure:

```text
-> Actual seconds

-> CPU speed

-> RAM size

-> Computer performance
```

Instead, it measures:

```text
Number of Operations
```

performed as input size increases.

---

## Example

```java
for(int i = 0; i < n; i++) {
    System.out.println(i);
}
```

Operations:

```text
n times
```

Time Complexity:

```text
O(n)
```

---

# What is Space Complexity?

## Definition

=> Space Complexity measures the amount of memory required by an algorithm as input size increases.

Memory includes:

```text
-> Variables

-> Arrays

-> Objects

-> Function Call Stack

-> Dynamic Memory
```

---

## Example

```java
int[] arr = new int[n];
```

Memory Required:

```text
n integers
```

Space Complexity:

```text
O(n)
```

# Asymptotic Notations

## Q: What are Asymptotic Notations?

=> Mathematical tools used to describe algorithm performance.

Types:

```text
1. Big O Notation

2. Omega Notation

3. Theta Notation
```

---

# Big O Notation

## Definition

=> Represents the Worst Case Complexity.

Example:

```java
for(int i = 0; i < n; i++) {
    System.out.println(i);
}
```

Complexity:

```text
O(n)
```

Meaning:

```text
Maximum growth rate.
```

---

# Omega Notation

## Definition

=> Represents Best Case Complexity.

Symbol:

```text
Ω
```

Example:

Linear Search

Searching first element:

```text
Ω(1)
```

---

# Theta Notation

## Definition

=> Represents Average or Exact Growth Rate.

Symbol:

```text
Θ
```

Example:

```text
Θ(n)
```

---

# Common Time Complexities

# O(1) — Constant Time

## Definition

Execution time remains constant.

Example:

```java
int[] arr = {10,20,30};

System.out.println(arr[1]);
```

Operations:

```text
Always 1 operation.
```

Complexity:

```text
O(1)
```

---

## Real Applications

```text
-> Array Access

-> Stack Top Access

-> Queue Front Access

-> HashMap Lookup (Average)
```

---

# O(log n) — Logarithmic Time

## Definition

Problem size reduces by half repeatedly.

Example:

Binary Search

```java
while(low <= high) {

    int mid = (low + high)/2;

    if(arr[mid] == key)
        return mid;

    else if(arr[mid] < key)
        low = mid + 1;

    else
        high = mid - 1;
}
```

Complexity:

```text
O(log n)
```

---

## Real Applications

```text
-> Binary Search

-> Database Indexing

-> Search Engines

-> Balanced Trees
```

---

# O(n) — Linear Time

## Definition

Operations grow directly with input size.

Example:

```java
for(int i = 0; i < n; i++) {

}
```

Complexity:

```text
O(n)
```

---

## Real Applications

```text
-> Array Traversal

-> Linear Search

-> Counting Elements

-> Data Processing
```

---

# O(n log n)

## Definition

Combination of:

```text
n × log n
```

Example:

Merge Sort

Quick Sort (Average Case)

```text
O(n log n)
```

---

## Real Applications

```text
-> Sorting Huge Datasets

-> Database Operations

-> Search Ranking Systems

-> Analytics Engines
```

---

# O(n²) — Quadratic Time

## Definition

Nested loops.

Example:

```java
for(int i = 0; i < n; i++) {

    for(int j = 0; j < n; j++) {

    }
}
```

Complexity:

```text
O(n²)
```

---

## Real Applications

```text
-> Bubble Sort

-> Selection Sort

-> Matrix Comparisons

-> Pattern Matching
```

---

# O(n³) — Cubic Time

Example:

```java
for(int i = 0; i < n; i++) {

    for(int j = 0; j < n; j++) {

        for(int k = 0; k < n; k++) {

        }
    }
}
```

Complexity:

```text
O(n³)
```

---

## Real Applications

```text
-> Matrix Multiplication

-> Scientific Simulations

-> Graph Algorithms
```

---

# O(2ⁿ) — Exponential Time

## Definition

Operations double for every new input.

Example:

Recursive Fibonacci

```java
fib(n) = fib(n-1) + fib(n-2)
```

Complexity:

```text
O(2ⁿ)
```

---

## Problem

Very slow for large inputs.

---

# O(n!)

## Definition

Factorial Growth.

Example:

Traveling Salesman Problem (Brute Force)

Complexity:

```text
O(n!)
```

---

## Example

```text
5! = 120

10! = 36,28,800
```

Extremely expensive.

---

# Rules for Calculating Time Complexity

# Rule 1: Ignore Constants

Example:

```java
for(int i = 0; i < 100*n; i++) {

}
```

Complexity:

```text
O(n)
```

Not:

```text
O(100n)
```

---

# Rule 2: Drop Lower Order Terms

Example:

```java
n² + n + 5
```

Complexity:

```text
O(n²)
```

---

# Rule 3: Sequential Loops Add

Example:

```java
for(int i=0;i<n;i++) {

}

for(int i=0;i<n;i++) {

}
```

Complexity:

```text
O(n + n)

O(2n)

O(n)
```

---

# Rule 4: Nested Loops Multiply

Example:

```java
for(int i=0;i<n;i++) {

    for(int j=0;j<n;j++) {

    }
}
```

Complexity:

```text
O(n²)
```

---


Complexity:



Complexity:

```text
O(n)
```

---



---

# Auxiliary Space

## Definition

Extra memory used by algorithm excluding input.

Example:

```java
int[] temp = new int[n];
```

Auxiliary Space:

```text
O(n)
```

---

# Recursive Space Complexity

Example:

```java
void fun(int n) {

    if(n == 0)
        return;

    fun(n - 1);
}
```

Call Stack:

```text
n calls
```

Space Complexity:

```text
O(n)
```

---

# Time vs Space Tradeoff

## Definition

Sometimes we use more memory to reduce execution time.

Example:

HashMap

```text
More Space

Less Time
```
