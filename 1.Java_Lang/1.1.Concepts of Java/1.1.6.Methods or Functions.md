# Introduction to Methods (Functions)

## What is a Function?

In Java, functions are called **Methods**.

A method is a reusable block of code that performs a specific task and executes only when it is called.

Java is a class-based language, therefore every method must be declared inside a class.

Example:

```java
class Demo {

    static void greet() {

        System.out.println("Hello Java");
    }

    public static void main(String[] args) {

        greet();
    }
}
```

Output:

```text
Hello Java
```

---

## Why Do We Need Methods?

Without methods:

```java
System.out.println(5 + 10);
System.out.println(20 + 30);
System.out.println(40 + 50);
```

Logic is repeated.

With methods:

```java
static int add(int a, int b) {

    return a + b;
}
```

Usage:

```java
add(5,10);
add(20,30);
add(40,50);
```

Benefits:

- Code Reusability
- Modularity
- Better Readability
- Easier Maintenance
- Easier Testing
- Reduced Complexity
- Better Organization

# Method Execution Flow

Example:

```java
class Demo {

    static void first() {

        System.out.println("First");
    }

    static void second() {

        System.out.println("Second");
    }

    public static void main(String[] args) {

        first();

        second();
    }
}
```

Execution Flow:

```text
JVM Starts
      ↓
main()
      ↓
first()
      ↓
Return to main()
      ↓
second()
      ↓
Return to main()
      ↓
Program Ends
```

Important Rule:

```text
After method execution,
control always returns to the caller.
```

# Parameters and Arguments

## Parameters

Variables declared inside method definition.

Example:

```java
int add(int a, int b)
```

Here:

```text
a and b
```

are parameters.

---

## Arguments

Actual values passed during method call.

Example:

```java
add(10,20);
```

Here:

```text
10 and 20
```

are arguments.

---

## Comparison

| Parameters                    | Arguments                 |
| ----------------------------- | ------------------------- |
| Defined in method declaration | Passed during method call |
| Placeholder values            | Actual values             |
| Variables                     | Data                      |

Example:

```java
add(int a, int b)
```

```java
add(10,20)
```

```text
a,b → Parameters

10,20 → Arguments
```

# Java Pass By Value

## Important Interview Concept

Java is always:

```text
Pass By Value
```

There is NO pass-by-reference in Java.

---

# Primitive Example

```java
class Demo {

    static void change(int x) {

        x = 100;
    }

    public static void main(String[] args) {

        int a = 10;

        change(a);

        System.out.println(a);
    }
}
```

Output:

```text
10
```

Reason:

```text
Copy of a is passed.
```

---

# Object Example

```java
class Student {

    String name;
}
```

```java
static void modify(Student s) {

    s.name = "Rahul";
}
```

```java
Student obj = new Student();

modify(obj);
```

Object changes.

Reason:

```text
Copy of reference is passed.
```

---


Method receives:


# Method Overloading

## Definition

Same method name.

Different parameter list.

Compiler decides which method to call.

This is called:

```text
Compile Time Polymorphism
```

---

## Example

```java
class MathUtil {

    int add(int a, int b) {

        return a + b;
    }

    int add(int a, int b, int c) {

        return a + b + c;
    }
}
```

---

## Example 2

```java
double add(double a, double b) {

    return a + b;
}
```

---

## Valid Overloading

Different:

- Number of Parameters
- Parameter Types
- Parameter Order

---

## Invalid Overloading

Wrong:

```java
int add(int a, int b)
```

```java
double add(int a, int b)
```

Only return type differs.

Compiler Error.

---

# Method Overloading Rules

| Allowed              | Not Allowed                |
| -------------------- | -------------------------- |
| Different Parameters | Different Return Type Only |
| Different Data Types | Same Signature             |
| Different Count      | Same Method                |

# Static Methods

## Definition

Belong to class.

No object required.

Called using:

```java
ClassName.methodName()
```

---

## Example

```java
class Demo {

    static void show() {

        System.out.println("Hello");
    }

    public static void main(String[] args) {

        Demo.show();
    }
}
```

---

## Real Examples

```java
Math.sqrt()
```

```java
Math.pow()
```

```java
Integer.parseInt()
```

---

## Characteristics

- Class Level
- Memory Efficient
- Shared by All Objects
- Can Be Called Directly

# Instance Methods

## Definition

Belong to object.

Require object creation.

---

## Example

```java

```

Call:


---

# Static vs Instance Method

| Static Method                  | Instance Method                           |
| ------------------------------ | ----------------------------------------- |
| Belongs to Class               | Belongs to Object                         |
| No Object Needed               | Object Needed                             |
| Shared by All Objects          | Separate for Every Object                 |
| Access Static Members Directly | Access Both Static and Non-Static Members |

# Recursion

## Definition

Method calling itself.

---

## Example

```java
int factorial(int n) {

    if(n == 1)
        return 1;

    return n * factorial(n - 1);
}
```

Call:

```java
factorial(5)
```

Execution:

```text
5 * factorial(4)
5 * 4 * factorial(3)
5 * 4 * 3 * factorial(2)
5 * 4 * 3 * 2 * factorial(1)
```

Result:

```text
120
```

---

# Components of Recursion

## Base Case

Stops recursion.

```java
if(n == 1)
    return 1;
```

---

## Recursive Call

Calls itself.

```java
factorial(n - 1)
```

---

Without Base Case:

```text
StackOverflowError
```
