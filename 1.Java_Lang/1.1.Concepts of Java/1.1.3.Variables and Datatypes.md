# Variables and Data Types in Java

---

# 1. Variables in Java

## What is a Variable?

A variable is a named memory location used to store data during the execution of a Java program.

Since Java is a **statically typed language**, every variable must be declared with a specific data type before it can be used.

Think of a variable as a container that stores a value.

Example:

```java
int age = 21;
```

Here:

- `int` → Data Type
- `age` → Variable Name
- `21` → Stored Value

---

## Why Do We Need Variables?

Variables help us:

- Store data
- Reuse data
- Modify data during execution
- Perform calculations
- Make programs dynamic

Without Variables:

```java
System.out.println(21);
System.out.println(21);
System.out.println(21);
```

With Variables:

```java
int age = 21;

System.out.println(age);
System.out.println(age);
System.out.println(age);
```

If the value changes, update only one place.

# Types of Variables in Java

Java provides three major types of variables based on scope.

---

## 1. Local Variables

Declared inside methods, constructors, or blocks.

Example:

```java
public class Demo {

    public static void main(String[] args) {

        int x = 10;
    }
}
```

Characteristics:

- Created when method starts
- Destroyed when method ends
- Stored in Stack Memory
- No default value
- Must be initialized before use

---

## 2. Instance Variables

Declared inside a class but outside methods.

Example:

```java
class Student {

    int age;
}
```

Characteristics:

- Belong to objects
- Stored in Heap Memory
- Each object gets its own copy
- Receive default values automatically

---

## 3. Static Variables

Declared using the static keyword.

Example:

```java
class Student {

    static String college = "ABC";
}
```

Characteristics:

- Belong to class
- Shared among all objects
- Only one copy exists
- Stored in Method Area

---

# Memory Representation

Example:

```java
Student s = new Student();
```

Memory:

```text
STACK                     HEAP

s ----------------->      Student Object
```

Reference Variable:

```java
s
```

Stored in Stack Memory.

Object:

```java
new Student()
```

Stored in Heap Memory.

---

# Scope of Variables

Scope determines where a variable can be accessed.

---

## Local Scope

```java
void display() {

    int x = 10;
}
```

Accessible only inside display().

---

## Class Scope

```java
class Student {

    int age;
}
```

Accessible throughout the object.

---

# Lifetime of Variables

| Variable Type | Lifetime                  |
| ------------- | ------------------------- |
| Local         | Until method ends         |
| Instance      | Until object is destroyed |
| Static        | Until program terminates  |

---

# Default Values of Variables

| Data Type      | Default Value |
| -------------- | ------------- |
| byte           | 0             |
| short          | 0             |
| int            | 0             |
| long           | 0L            |
| float          | 0.0f          |
| double         | 0.0           |
| char           | '\u0000'      |
| boolean        | false         |
| Reference Type | null          |

Note:

Local variables do not receive default values.

# 2. Data Types in Java

## What is a Data Type?

A Data Type specifies:

- What kind of data can be stored
- Memory required
- Operations allowed on that data

Example:

```java

```

Here:

```java
int
```

defines the type of data.

```text

      └───────────── Non-Primitive
```

# Primitive vs Non-Primitive

| Feature      | Primitive    | Non-Primitive   |
| ------------ | ------------ | --------------- |
| Stores       | Actual Value | Reference       |
| Memory       | Fixed        | Dynamic         |
| Methods      | No           | Yes             |
| Null Allowed | No           | Yes             |
| Performance  | Faster       | Slightly Slower |

# Advantages of Java Data Types

## Type Safety

Compile-time error detection.

Wrong:

```java
int age = "Sahil";
```

Compiler Error.

---

## Better Performance

Primitives consume less memory.

Example:

```java
int x = 10;
```

Directly stored.

---

## Predictable Memory Usage

Every primitive type has a fixed size.

---

# Disadvantages

## Rigid Type System

Cannot change variable type dynamically.

Wrong:

```java
int x = 10;
x = "Hello";
```

Compiler Error.

---

## Primitive vs Object Separation

Primitives are not objects.

Example:

```java
int x = 10;
```

Cannot use object methods.

To store inside collections:

```java
ArrayList<Integer>
```

Java automatically converts:

```java
int
```

to

```java
Integer
```

This process is called Autoboxing.

# Summary

- Variables are named memory locations.
- Java is statically typed.
- Variables store data during execution.
- Variables are Local, Instance, or Static.
- Data Types define the kind of data stored.
- Java provides 8 Primitive Data Types.
- Non-Primitive types store references.
- Primitive types are faster and memory efficient.
- Non-Primitive types provide flexibility and object-oriented features.
- Understanding Variables and Data Types is the foundation of all Java programming.
