# 1. Introduction to Scope in Java

## What is Scope?

Scope refers to the region of a program where a variable, method, or object reference can be accessed.

In simple words:

```text
Scope determines:

1. Where a variable can be used.
2. Where a variable is visible.
3. How long a variable lives in memory.
4. When a variable gets destroyed.
```

Every variable in Java has:

- Visibility
- Accessibility
- Lifetime

These three properties are controlled by its scope.

---

# Why Scope is Important?

Scope helps:

- Prevent variable conflicts
- Improve code readability
- Improve security
- Reduce memory usage
- Improve maintainability
- Control accessibility

Without scope, every variable would be accessible everywhere, making programs difficult to manage.

# 3. Class Scope (Static Scope)

## What is Class Scope?

Variables declared using:

```java
static
```

inside a class but outside methods belong to the class itself.

Example:

```java
class Student {

    static String college = "Parul University";
}
```

---

## Memory Location

Stored in:

```text
Method Area
```

inside JVM memory.

Only one copy exists.

Memory:

```text
Method Area

college = "Parul University"
```

---

## Characteristics

- Shared by all objects
- Loaded once
- Accessible without object creation
- Exists until JVM shuts down

---

## Example

```java
class Student {

    static String college = "Parul University";
}

public class Demo {

    public static void main(String[] args) {

        Student s1 = new Student();
        Student s2 = new Student();

        System.out.println(Student.college);
    }
}
```

Output:

```text
Parul University
```

Only one copy exists.

---

## Lifetime

Born:

```text
When class is loaded
```

Destroyed:

```text
When JVM terminates
```

---

# 4. Instance Scope

## What is Instance Scope?

Variables declared inside a class but outside methods without:

```java
static
```

are called instance variables.

Example:

```java
class Student {

    String name;
}
```

---

## Memory Location

Stored inside:

```text
Heap Memory
```

Each object gets its own copy.

---

## Example

```java
class Student {

    String name;
}

public class Demo {

    public static void main(String[] args) {

        Student s1 = new Student();
        Student s2 = new Student();

        s1.name = "Sahil";
        s2.name = "Rahul";

        System.out.println(s1.name);
        System.out.println(s2.name);
    }
}
```

Output:

```text
Sahil
Rahul
```

---

## Memory Representation

```text
STACK

s1 ───────────────┐
                  │
s2 ───────────────┼────────────┐
                  │            │
                  ▼            ▼

HEAP

Student Object      Student Object

name="Sahil"        name="Rahul"
```

---

## Characteristics

- Belongs to objects
- Separate copy for each object
- Stored in Heap
- Accessed through object references

---

## Lifetime

Born:

```text
Object Creation
```

Destroyed:

```text
When Object is Garbage Collected
```

---

# 5. Method Scope (Local Scope)

## What is Method Scope?

Variables declared inside a method are local variables.

Example:

```java
void display() {

    int x = 10;
}
```

---

## Visibility

Accessible only inside:

```java
display()
```

method.

Outside method:

```java
x
```

does not exist.

---

## Example

```java
class Demo {

    void show() {

        int age = 20;

        System.out.println(age);
    }
}
```

Valid:

```java
show()
```

Invalid:

```java
System.out.println(age);
```

outside method.

---

## Memory Location

Stored inside:

```text
Stack Frame
```

---

## Lifetime

Created:

```text
Method Invocation
```

Destroyed:

```text
Method Completion
```

---

# 6. Block Scope

## What is Block Scope?

Variables declared inside:

```java
{}
```

are block variables.

Examples:

```java
if
for
while
switch
try
catch
```

blocks.

---

## Example

```java
if(true) {

    int x = 10;

    System.out.println(x);
}
```

Outside:

```java
System.out.println(x);
```

Compiler Error.

---

## Memory Representation

```text
Method Scope
│
├── Block Scope
│
└── x = 10
```

After block ends:

```text
x destroyed
```

---

## Example

```java
for(int i = 0; i < 5; i++) {

}
```

Here:

```java
i
```

exists only inside loop.

# 8. Variable Shadowing

## What is Variable Shadowing?

Occurs when:

```java
Local Variable
```

and

```java
Instance Variable
```

have the same name.

Example:

```java
class Student {

    String name;

    Student(String name) {

        name = name;
    }
}
```

Problem:

```text
Parameter shadows instance variable
```

Both names refer to parameter.

Instance variable remains unchanged.

---

# 9. Introduction to this Keyword

## What is this?

The keyword:

```java
this
```

is a special reference variable.

It refers to:

```text
Current Object
```

currently executing the code.

---

## Meaning

Suppose:

```java
Student s1 = new Student();
```

Inside constructor:

```java
this
```

means:

```java
s1
```

reference.

---

# 10. Why this is Needed?

Example:

```java
class Student {

    String name;

    Student(String name) {

        name = name;
    }
}
```

Problem:

```text
Parameter hides instance variable
```

Both names refer to:

```java
name
```

parameter.

Instance variable never changes.

---

# Correct Version

```java
class Student {

    String name;

    Student(String name) {

        this.name = name;
    }
}
```

Meaning:

```java
this.name
```

→ Instance Variable

```java
name
```

→ Constructor Parameter

# 11. Common Uses of this

## 1. Resolving Variable Shadowing

```java
this.name = name;
```

Most common use.

---

## 2. Calling Current Object Method

```java
this.display();
```

Example:

```java
class Demo {

    void display() {

        System.out.println("Hello");
    }

    void show() {

        this.display();
    }
}
```

---

## 3. Calling Another Constructor

```java
this(...)
```

Example:

```java
class Student {

    Student() {

        this("Unknown");
    }

    Student(String name) {

    }
}
```

Called:

```text
Constructor Chaining
```

---

## 4. Passing Current Object

Example:

```java
method(this);
```

Passes current object reference.

---

## 5. Returning Current Object

Example:

```java
return this;
```

Used in:

```text
Method Chaining
Builder Pattern
Fluent APIs
```

---

# 12. When this Cannot Be Used

Inside:

```java
static methods
```

Example:

```java
static void show() {

    this.name = "Sahil";
}
```

Compiler Error.

Reason:

```text
Static methods belong to class.
No object exists.
```

Therefore:

```java
this
```

cannot be used.
